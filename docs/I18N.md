# 🌐 Sistema de Internacionalización (i18n)

El Pole Bot ahora soporta **múltiples idiomas** de forma sencilla y eficiente.

## 📋 Idiomas Soportados

- 🇪🇸 **Español** (default)
- 🇬🇧 **English**

## ⚙️ Configuración

### Para Administradores de Servidores

1. Usa el comando `/settings`
2. Selecciona "Idioma" en el menú desplegable
3. Elige entre Español 🇪🇸 o English 🇬🇧
4. ¡Listo! Todos los mensajes del bot cambiarán al idioma seleccionado

**Nota:** Solo los administradores pueden cambiar el idioma del servidor.

## 🔧 Para Desarrolladores

### Uso Básico

El sistema i18n usa un diccionario centralizado en `utils/i18n.py` y una función helper `t()` para traducir strings.

#### Importar la función:

```python
from utils.i18n import t, get_language_name
```

#### Usar traducciones:

```python
# Traducción simple
mensaje = t('settings.title', guild_id)
# ES: "⚙️ Configuración del Pole Bot"
# EN: "⚙️ Pole Bot Settings"

# Traducción con variables
mensaje = t('pole.critical', guild_id, minutes=5)
# ES: "⚡ ¡POLE CRÍTICO! En solo **5 minutos**. Eres una máquina."
# EN: "⚡ CRITICAL POLE! Only **5 minutes**. You're a machine."

# Forzar idioma específico
mensaje = t('common.yes', lang='en')
# "Yes"
```

### Estructura de Keys

Las keys usan notación de punto para organización:

```
<categoría>.<subcategoría>.<elemento>
```

Ejemplos:

- `cmd.settings.desc` - Descripción del comando /settings
- `pole.critical` - Mensaje de pole crítico
- `settings.title` - Título del embed de settings
- `common.yes` - String común "Sí"/"Yes"

### Añadir Nuevas Traducciones

Edita `utils/i18n.py` y añade la key en ambos idiomas:

```python
TRANSLATIONS = {
    'es': {
        'mi.nueva.key': 'Texto en español con {variable}',
    },
    'en': {
        'mi.nueva.key': 'Text in English with {variable}',
    }
}
```

### Formateo con Variables

Usa `{nombre_variable}` en el string y pásala como argumento:

```python
# En i18n.py
'es': {
    'profile.stats': '💰 {points} pts • 🏁 {poles} poles'
}

# En el código
texto = t('profile.stats', guild_id, points=1250.5, poles=50)
# "💰 1250.5 pts • 🏁 50 poles"
```

### Obtener Idioma del Servidor

```python
from utils.database import Database

db = Database()
lang = db.get_server_language(guild_id)  # 'es' o 'en'
```

### Cambiar Idioma (programáticamente)

```python
db.set_server_language(guild_id, 'en')
```

## 📚 Convenciones

### ✅ Buenas Prácticas

1. **Keys descriptivas:** `pole.critical` en vez de `pc`
2. **Agrupación lógica:** Usa categorías (`cmd.*`, `pole.*`, `settings.*`)
3. **Mantener personalidad:** Las traducciones deben tener CARISMA, no ser corporativas
4. **Variables claras:** Usa nombres descriptivos (`{user}`, `{points}`, no `{x}`, `{y}`)

### ❌ Evitar

1. **Keys hardcoded:** Nunca usar strings directos, siempre pasar por `t()`
2. **Traducciones literales:** Adapta el tono y flow de cada idioma
3. **Emojis duplicados:** Los emojis van en el string traducido, no fuera
4. **Mezclar idiomas:** Una key debe estar en todos los idiomas soportados

## 🎯 Ejemplo Completo

**Antes (sin i18n):**

```python
embed = discord.Embed(
    title="🏆 RANKING LOCAL - Personas",
    description=f"Servidor: {guild.name}"
)
embed.add_field(
    name="🥇 Primer Puesto",
    value=f"💰 {points} pts • 🏁 {poles} poles"
)
await interaction.response.send_message(embed=embed)
```

**Después (con i18n):**

```python
from utils.i18n import t

embed = discord.Embed(
    title=t('leaderboard.title', guild_id,
            scope=t('leaderboard.scope.local', guild_id),
            type=t('leaderboard.type.people', guild_id)),
    description=t('leaderboard.server_desc', guild_id, server=guild.name)
)
embed.add_field(
    name=t('leaderboard.first_place', guild_id),
    value=t('leaderboard.stats', guild_id, points=points, poles=poles)
)
await interaction.response.send_message(embed=embed)
```

## 🔍 Qué se Traduce

### ✅ Traducible dinámicamente:

- Contenido de mensajes
- Títulos y descripciones de embeds
- Labels de botones y selects
- Placeholders de inputs
- Mensajes de error/éxito
- Descripciones de comandos (al registrar)

### ❌ NO traducible (limitación de Discord):

- Nombres de comandos (`/settings` siempre es `/settings`)
- Nombres de parámetros (`usuario` siempre es `usuario`)
- Valores internos (`"critical"`, `"global"`, etc.)

## 🎬 La Esencia en Inglés

**Importante:** Las traducciones en inglés también tienen **personalidad**:

```python
# ✅ BIEN - Con flow
'pole.critical': '⚡ CRITICAL POLE! Only **{minutes} minutes**. You\'re a machine.'
'pole.marranero': '🐷 Late pole... Better late than never.'

# ❌ MAL - Corporativo
'pole.critical': 'You have achieved a critical pole in {minutes} minutes.'
'pole.marranero': 'Pole registered with delay.'
```

No es traducción literal, es **traducción con carisma**. 🔥
