# i18n - Internacionalizacion

Pole Bot soporta idioma por servidor.

## Idiomas

- `es` (default)
- `en`

## Donde vive

- Traducciones: `utils/i18n.py`
- Idioma por servidor: tabla `servers.language` (schema v6+)

Schema actual del proyecto: v7.

## Uso en codigo

```python
from utils.i18n import t

msg = t('settings.title', guild_id)
msg2 = t('pole.critical', guild_id, minutes=5)
```

## Reglas practicas

- No hardcodear mensajes de usuario en cogs.
- Toda key nueva debe existir en `es` y `en`.
- Mantener placeholders consistentes entre idiomas.

## Variaciones aleatorias

Algunas keys pueden ser lista en lugar de string.

`t()` elige una opcion aleatoria automaticamente para dar variedad en mensajes frecuentes.

## Scripts de validacion (repo actual)

- `scripts/check_missing_translations.py`
- `scripts/check_placeholders.py`

Ejemplo:

```bash
python scripts/check_missing_translations.py
python scripts/check_placeholders.py
```

## Nota

Los comandos debug no estan pensados para localizacion completa de producto; se prioriza i18n en experiencia de usuario final.
